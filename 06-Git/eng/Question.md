# Git & GitHub Interview Questions

1. What is Git and why is it used?
2. What is the difference between Git and GitHub?
3. What is a Git repository?
4. What is the difference between `git init` and `git clone`?
5. What are the three stages (areas) in Git?
6. What is the difference between `git add` and `git commit`?
7. What is the difference between `git pull` and `git fetch`?
8. What is the difference between `git merge` and `git rebase`?
9. What is a merge conflict and how do you resolve it?
10. What is a Git branch and why do we use branches?
11. How do you create, switch, and delete branches in Git?
12. What is `HEAD` in Git?
13. What is a detached HEAD state?
14. What is the difference between `git reset`, `git revert`, and `git checkout`?
15. What are the three modes of `git reset` (soft, mixed, hard)?
16. What is `git stash` and when would you use it?
17. What is a `.gitignore` file and why is it important?
18. What is the difference between a fast-forward merge and a three-way merge?
19. What is `git cherry-pick` and when is it useful?
20. What is `git bisect` and how does it help debug issues?
21. What is the difference between `git diff`, `git diff --staged`, and `git diff HEAD`?
22. What are Git tags and what is the difference between lightweight and annotated tags?
23. What is a pull request (PR) and how does it fit into a team workflow?
24. What is a fork in GitHub and how does it differ from a branch?
25. What are GitHub Actions and how are they used for CI/CD?
26. What is a Git hook? Name some common hooks and their use cases.
27. How do you squash commits and why would you do it?
28. What is `git reflog` and how can it help recover lost commits?
29. What is the difference between `git rm` and simply deleting a file?
30. What is a Git submodule and when would you use one?
